## Get Started in 5 Minutes

This guide will help you quickly set up and customize the Astro portfolio template. Follow these steps to have your portfolio running locally.

<Steps>
  <Step title="Clone the repository">
    Open your terminal and clone the repository to your local machine:

    ```bash
    git clone https://github.com/vaibhav227/portfolio
    cd portfolio
    ```

    <Note>
      You can also fork the repository first if you want to make it your own on GitHub.
    </Note>
  </Step>

  <Step title="Install dependencies">
    This project uses **pnpm** as the package manager. Install all dependencies:

    ```bash
    pnpm install
    ```

    <Warning>
      Make sure you have pnpm 8 or higher installed. If not, install it with `npm install -g pnpm`
    </Warning>

    The template includes these key dependencies:
    - `astro` - The core framework
    - `@astrojs/react` - React integration
    - `@astrojs/tailwind` - Tailwind CSS integration
    - `@astrojs/mdx` - MDX support
    - `tailwindcss` - Utility-first CSS
    - `typescript` - Type checking
  </Step>

  <Step title="Start the development server">
    Launch the local development server:

    ```bash
    pnpm run dev
    ```

    The server will start at `http://localhost:4312`. Open this URL in your browser to see your portfolio.

    <Tip>
      You can also use `pnpm start` as an alias for `pnpm run dev`
    </Tip>
  </Step>

  <Step title="Customize your profile data">
    Edit `src/content/profileData.ts` to add your personal information:

    ```typescript src/content/profileData.ts
    export const PROFILE = {
      site: {
        SEO: {
          title: "Your Name - Your Tagline",
          description: "Your professional description for SEO",
        },
      },
      timezone: "America/New_York", // Your timezone
      language: "en-US",
      firstName: "Your",
      name: "Your Name",
      headLine: "Your tagline",
      headLine2: "Your detailed description about what you do...",
      website: "https://yourwebsite.com",
      repo: "https://github.com/yourusername/portfolio",
      shortIntros: [
        "💻 Software Engineer",
        "🚀 Building amazing things",
        "☘️ Open source contributor",
      ],
      skills: [
        "JavaScript",
        "TypeScript",
        "React",
        "Astro",
        // Add your skills
      ],
      links: {
        github: "https://github.com/yourusername",
        mail: "mailto:your@email.com",
        linkedin: "https://linkedin.com/in/yourusername",
      },
      studies: [
        {
          title: "Your Degree",
          level: "Bachelor",
          score: "GPA or percentage",
          endYear: "2023",
        },
      ],
    };
    ```

    This single file controls all your personal information displayed across the site.
  </Step>

  <Step title="Update site configuration">
    Edit `astro.config.ts` to set your website URL:

    ```typescript astro.config.ts
    import { defineConfig } from "astro/config";
    import react from "@astrojs/react";
    import tailwind from "@astrojs/tailwind";
    import mdx from "@astrojs/mdx";

    export default defineConfig({
      site: "https://yourwebsite.com", // Change this to your domain
      integrations: [
        react(),
        tailwind({
          applyBaseStyles: false,
        }),
        mdx(),
      ],
    });
    ```

    Also update the `site` field in `package.json` to match your domain.
  </Step>

  <Step title="Add your content">
    The template uses Astro Content Collections for managing projects, blog posts, and experiences. Here's how to add content:

    ### Add a Project

    Create a new `.mdx` file in `src/content/projects/`:

    ```mdx src/content/projects/my-project.mdx
    ---
    title: "My Awesome Project"
    startDate: 2024-01-15
    description: "A brief description of what this project does"
    image:
      url: "/images/my-project.png"
      alt: "Project screenshot"
    tags: ["React", "TypeScript", "Tailwind"]
    ---

    ## Overview

    Detailed description of your project goes here.

    ## Features

    - Feature 1
    - Feature 2
    - Feature 3

    ## Tech Stack

    Built with React, TypeScript, and Tailwind CSS.
    ```

    ### Add a Blog Post

    Create a new `.mdx` file in `src/content/posts/`:

    ```mdx src/content/posts/my-first-post.mdx
    ---
    title: "My First Blog Post"
    startDate: 2024-03-04
    description: "An introduction to my blog"
    tags: ["webdev", "astro"]
    ---

    Your blog content goes here with full MDX support!
    ```

    ### Add an Experience

    Create a new `.mdx` file in `src/content/experiences/`:

    ```mdx src/content/experiences/current-job.mdx
    ---
    title: "Software Engineer"
    company: "Amazing Company"
    startDate: 2023-01-01
    tags: ["React", "Node.js"]
    ---

    Description of your role and responsibilities.
    ```

    <Note>
      All dates should be in `YYYY-MM-DD` format. The `endDate` field is optional for experiences and indicates current positions when omitted.
    </Note>
  </Step>
</Steps>

## Verify Everything Works

Your portfolio should now be running with your customizations! Check that:

✅ The homepage displays your name and introduction
✅ Your social links work correctly
✅ Your projects, posts, and experiences appear
✅ Dark/light mode toggle works
✅ Navigation between pages functions properly

## Build for Production

When you're ready to deploy, build your site:

```bash
pnpm run build
```

This creates an optimized production build in the `dist/` directory.

To preview the production build locally:

```bash
pnpm run preview
```
